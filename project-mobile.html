Process

Designing for Every Context

01

Defined user roles and contexts

Mapped out three primary user groups — facility managers, field staff, and administrators — understanding how each interacts with documents and tasks across different devices and locations.

02

Built an adaptive design system

Created a component library that intelligently adapts across breakpoints — same components, different compositions — ensuring consistency without designs feeling like a shrunken desktop on mobile.

03

Optimised for real-world use

Validated designs in context — field staff using iPads on-site, managers reviewing reports on desktop — iterating until every critical workflow was fast, clear, and error-proof.
